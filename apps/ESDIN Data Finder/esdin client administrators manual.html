Introduction

The ESDIN client is a Javascript web client. Its purpose is to provide interaction with ESDIN services (View Services and Download Services).

This manual provides instructions for those in charge of maintenance of the client. Changing client settings is done by making changes in one of the configuration files. These are the configuration files:

  • layers.js
  • downloadServices.js
  • testAreas.js
  • configuration.js

All these files are javascript files. However, no knowledge of the javascript language is needed. The javascript syntax rules need to be obeyed, but it should be possible to change settings by copy/paste/cut operations, optionally followed by changing text strings (enclosed in double quotes (")).

Managing basic settings

Basic settings are found in the file configuration.js. Among other things, the following settings are made here:

  • The name of the client
  • Often used URLs
  • Definition of Service Providers
  • Definition of themes and feature types

Each setting is headed by a comment explaining what the setting does.

Managing View Services

View Services are configured in layers.js. Each View Service shoudl be configured as an OpenLayers WMS layer. Existing layer definitions can be used as templates for new layers. Values that have been set in configuration.js may be used here, but that is not necessary.

Managing Download Services

Download Services are configure in downloadServices.js.

Managing test areas

Download Services are configure in testAreas.js.

Managing help pages

Help pages are avialable to the user through the help menu. The names of the help files correspond with the menu items. Help files are plain HTML files, they can be edited with any HTML editor.
