Introduction

The ESDIN client is a Javascript web client. Its purpose is to provide interaction with ESDIN services (View Services and Download Services).

Design guidelines

We have tried to follow the following guidlines:

  1. It should be easy to add or change WMSs or WFSs for non-developers
  2. It should be easy to edit the help files

Prerequisitites

The following libraries are used. They are all referenced in index.html:

  • OpenLayers (download: http://trac.osgeo.org/openlayers/wiki/HowToDownload)
  • GeoExt (download: http://www.geoext.org/downloads.html)
  • ExtJS (download: http://www.sencha.com/products/extjs/download/ orhttp://www.sencha.com/learn/Ext_Version_Archives)
  • Proj4Js (download: http://trac.osgeo.org/proj4js/wiki/Download)

Search by name

The 'search for a geographical name' functionality depends on a database with names data. See data_preperation.sql for the process of preparing the database, using PostgreSQL and PostGIS. The file complete.php should be runnable on the local web server.

Downloading GML from WFSs

This depends on an adaptation of the web server that hosst the client. See httpd.conf.fragment for the extra configuration needed for the Apache HTTP server.
